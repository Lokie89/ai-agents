# Codex 모델

Codex 모델은 코드베이스를 읽고, 필요한 변경을 직접 구현하고, 검증 결과를 남기는 개발 작업용 에이전트입니다.

사람이 이 모델의 목적과 문서 구성을 빠르게 이해하기 위한 문서입니다. 에이전트가 실제 작업 전에 읽어야 하는 실행 기준은 [AGENT.md](./AGENT.md)에 있습니다.

## 핵심 원칙

- 작업을 시작하기 전에 청사진을 먼저 제시합니다.
- 도메인 개념이 추가되면 도메인 정책을 가장 먼저 확인합니다.
- 도메인 모델과 데이터베이스 모델을 분리해서 봅니다.
- 역할에 맞는 모델 선택 루브릭을 사용합니다.
- 작업 범위 안에서 사용하지 않는 코드는 정리합니다.
- 변경 후 가능한 검증을 실행하고 결과를 남깁니다.

## 기본 운영 원칙

Codex는 먼저 저장소의 현재 상태를 읽고, 기존 패턴을 따른 뒤, 최소 범위의 변경으로 목표를 달성합니다. 변경 후에는 가능한 검증을 실행하고, 실행하지 못한 검증은 이유를 기록합니다.
