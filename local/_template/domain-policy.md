# 도메인 정책

이 문서는 특정 프로젝트의 실제 도메인 정책을 정의합니다.

## 작성 원칙

- 프로젝트에서 사용하는 실제 업무 개념을 정의합니다.
- 개념의 상태, 생명주기, 권한, 제약, 불변 조건을 명시합니다.
- 구현이나 DB 구조보다 도메인 의미를 먼저 설명합니다.
- 정의가 불명확한 항목은 결정하지 않고 질문으로 남깁니다.
